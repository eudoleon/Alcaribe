Speed Up your Odoo POS To Load It Up To 80 Times Faster

The one-stop solution that makes POS Data handling much more efficient for you. The module reduces POS loading time from minutes to seconds. The products and customers are now loaded instantly on the Odoo POS session.

More than 100,000 products and customers loaded in just 8 seconds in Odoo POS Shop. Moreover, the module supports cache and uses it to load POS faster.

Odoo & MongoDB - Integrated To Make Odoo POS More Efficient
The Module uses MongoDB to Sync and Upload data to Odoo POS. Since data is loaded using MongoDB server instead of Odoo Server. Hence, the load dramatically reduces on the Odoo POS. MongoDB works independently from your Odoo server, so there is no load on your Odoo. The result is a visible decrease in loading time.

The Odoo POS data, including various product and customer fields, are directly loaded from MongoDB instead of the Odoo server.

If any data such as price or price list is changed, it is automatically synced to MongoDB using Cron. Or the same can be updated to MongoDB manually with a click of a button.

No MongoDB? Then Use Your Local DB to Make POS Faster
Another Option provided for the users is to use their Local Odoo Database to load the POS Shop faster. The POS data is consolidated and stored at one place in the local DB speeding up the Odoo POS load time. The differences in product or customer data can be synced to this DB automatically or manually with a button click. In the same way, MongoDB can also be configured.

Syncing Data To The Odoo POS

  • Admin can choose to sync data in real-time in the Odoo backend.
  • Choose to sync changed data on page reload.
  • Also, the admin can choose to notify when changes occur.

Cache Support Further Helps With Faster Loading

  • The module also offers cache support which helps POS load even faster.
  • The data is cached with the browser when POS is Loaded, so next time the data is picked directly from the browser.

Picking The Right Database For Your Odoo POS

  • You can select to use a local DB for storing Odoo POS data.
  • Or you can choose MongoDB for storing your POS data.
  • For using MongoDB, the admin has to provide host and port links.

Load Products Faster In The Odoo POS

  • The module allows you to load products 80 times faster.
  • Load 100 thousand customers and products within seconds.
  • The module reduces the load time from minutes to seconds in Odoo.
